# Release and Sustainability

## Maintainer Expectations

- assign at least one owner for data, docs, backend, frontend, and ML service
- review open dataset and model issues on a regular cadence
- record breaking changes and data refreshes in release notes

## Recommended Cadence

- documentation review: monthly
- dependency review: monthly
- data source audit: quarterly
- model re-evaluation: quarterly or on major data updates

## Release Checklist

1. Docs build passes.
2. Tests pass.
3. Dataset manifests updated.
4. Model version metadata updated.
5. Changelog written.
6. Public site and Hugging Face wording aligned.

## Support Boundaries

This repository should explicitly state:
- whether the platform is research-only
- whether predictions are not for clinical decision making
- what support is available to external users
