# Technical Specification

## Goal

Provide the versioned technical specification needed to make the platform reproducible and maintainable.

## Required Implementation Documents

### Ingestion spec
- exact source dataset files or APIs
- version/retrieval metadata
- schema validation rules

### Harmonization spec
- identifier precedence
- matching heuristics
- manual curation rules
- audit table schema

### Modeling spec
- feature set definition
- split strategy
- baseline models
- evaluation outputs
- artifact versioning

### Serving spec
- frontend-backend interaction
- backend-prediction-service interaction
- request and response schemas
- timeout, retry, and error behavior
